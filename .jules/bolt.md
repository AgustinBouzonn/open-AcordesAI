## 2024-05-18 - [React Router Inline Component Recreation]
**Learning:** Defining view components as inline render functions (e.g., `renderHome()`) inside a parent component (`AppContent`) and passing their evaluation to React Router's `element` prop (`element={renderHome()}`) causes the inline function to execute on every single render of the parent, regardless of the active route. This executes heavy logic (like `localStorage` accesses in `getFavoriteSongsFull()` and `getHistorySongsFull()`) and recreates large unmounted VDOM trees unnecessarily on state changes (e.g., typing in a search bar).
**Action:** Always extract route view components into standalone React components defined completely outside of the parent component's scope. Pass them to React Router as standard JSX instances (`element={<HomeView />}`). This prevents component re-creation, unnecessary VDOM evaluation on inactive routes, and execution of heavy logic when parent state changes.
