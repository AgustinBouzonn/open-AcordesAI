## 2024-05-19 - Optimize SQL correlated subqueries with LATERAL joins
**Learning:** In PostgreSQL, having multiple separate correlated subqueries in the SELECT list (like `(SELECT AVG(score) FROM ratings WHERE song_id = s.id)` and `(SELECT COUNT(*) FROM ratings WHERE song_id = s.id)`) requires multiple independent scans of the target table per row.
**Action:** Consolidate these multiple subqueries into a single `LEFT JOIN LATERAL` block (e.g., `LEFT JOIN LATERAL (SELECT AVG(score), COUNT(*) FROM ratings WHERE song_id = s.id) r ON true`). This performs a single scan per row to calculate all required aggregations simultaneously, significantly improving query efficiency. Remember to use `COALESCE(r.rating, 0)` for `AVG` since it returns NULL for empty groups, while `COUNT` naturally returns 0 but we want to retain it properly as well.
