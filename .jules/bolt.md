## 2024-05-14 - Inline React Router Components Re-Render Trap
**Learning:** Defining React Router view components (e.g., `HomeView`, `SearchView`, `FavoritesView`, `HistoryView`) as inline functions inside a main component (like `AppContent`) forces React to re-create and execute their render logic on every unrelated state change. In `App.tsx`, because components like `renderFavorites` called `getFavoriteSongsFull()` (which accesses `localStorage`), typing in the search bar caused costly, redundant `localStorage` reads.
**Action:** Always extract route view components outside the main App/Router component. Wrap them in `React.memo()` and pass stable callbacks (via `useCallback`) to avoid unnecessary re-renders and costly operations (like `localStorage` access or deep prop comparisons) on unrelated state changes.
