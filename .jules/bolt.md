## 2023-11-20 - React Router Inline Functions Performance Bottleneck
**Learning:** Using inline function calls (e.g., `<Route path="/" element={renderHome()} />`) directly in the `element` prop of React Router components causes severe performance regressions. Every time the parent component re-renders (e.g., due to typing in a search bar), all inline functions for every route are executed simultaneously, even for inactive routes. This triggers unnecessary heavy computations, such as synchronous object reconstruction and DOM element creation, blocking the main thread.
**Action:** Always extract route views into standalone React components (e.g., `const HomeView = () => ...`) and use the component instance syntax (`<Route path="/" element={<HomeView />} />`). This ensures that only the active route's component is evaluated and that React can efficiently reconcile differences.
