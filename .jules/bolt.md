
## 2024-05-24 - Inline React Router Views Cause O(N) LocalStorage Reads
**Learning:** Defining React Router view components as inline functions (e.g., `renderFavorites()`) inside the main `AppContent` component and passing them directly to `<Route element={renderFavorites()} />` forces them to be evaluated on every single re-render of `AppContent`—even when the route doesn't match! Because `renderFavorites` and `renderHistory` perform synchronous `localStorage` reads and JSON parsing, every keystroke in the search bar causes O(N) unnecessary disk reads and parsing operations, severely blocking the main thread.
**Action:** Always extract React Router views into standalone, memoized components (`React.memo`) completely outside the parent component, and pass necessary state/callbacks via stable references (`useCallback`). This ensures views are only rendered when their route is active, and prevents unrelated state changes (like typing a search query) from triggering expensive operations in inactive views.
