## 2023-10-27 - [React Router Component Re-creation]
**Learning:** React Router view components passed to the `element` prop of `<Route>` should not be defined as inline render functions (e.g., `const renderHome = () => (...)`) or inline components inside a parent component. Doing so causes the components to be recreated completely on every render of the parent component, defeating React's diffing algorithm and causing performance bottlenecks. This is especially true if the parent manages a lot of changing state (like search queries, loading state).
**Action:** Always extract route components to standalone static components defined completely outside the parent router component, pass any required parent state as props, and consider wrapping them in `React.memo()` to skip rendering inactive routes entirely.
