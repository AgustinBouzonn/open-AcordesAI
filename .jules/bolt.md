## 2024-05-24 - Inline render functions in React Router Route elements
**Learning:** Defining view components as inline functions (e.g., `renderFavorites()`) inside a parent component and passing their invocation to a React Router `element` prop (e.g., `<Route element={renderFavorites()} />`) is a severe anti-pattern. This causes the function to be evaluated on EVERY render of the parent component, even when the route is not active. In this application, it caused `getFavoriteSongsFull()` and `getHistorySongsFull()` to run synchronously on every keystroke in the search bar.
**Action:** Always extract route view components completely outside the parent component and pass them as standard JSX elements (e.g., `<Route element={<FavoritesView />} />`) so React Router can control their rendering lifecycle efficiently.
