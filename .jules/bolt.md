## 2025-01-20 - PostgreSQL Subquery Consolidation via LATERAL Joins
**Learning:** In the PostgreSQL backend, using multiple correlated subqueries in the `SELECT` clause (like fetching AVG rating, COUNT rating, etc.) scans the dependent table multiple times per row in the outer query. While `LATERAL` joins also execute per-row, placing all these aggregations in a single `LEFT JOIN LATERAL` subquery consolidates these operations into a single scan of the dependent table per row, significantly reducing overhead.
**Action:** Replace multiple correlated subqueries on the same dependent table in `SELECT` and `ORDER BY` with consolidated `LEFT JOIN LATERAL` statements to reduce redundant table scans. Note that `AVG()` without records yields NULL and may need `COALESCE`, but `COUNT()` yields 0 natively.
