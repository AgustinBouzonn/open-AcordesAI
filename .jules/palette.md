## 2025-03-05 - Instrument Selector Group and Complex Interactive Controls Accessibility
**Learning:** In highly interactive custom toolbars like the SongViewer controls (with instrument toggle buttons, font size adjustments, and autoscroll features), individual buttons often lack critical context for screen reader users and keyboard navigators. Applying `role="group"` with an `aria-label` to grouping containers (like instrument selectors) and explicitly managing `aria-pressed` or `aria-expanded` states on custom toggle buttons provides necessary semantic meaning that standard `onClick` handlers do not supply natively. Additionally, relying on `hover` styles alone without explicit `focus-visible` styles makes these complex interfaces inaccessible to keyboard-only users in dark mode themes.
**Action:** When creating custom toolbars or toggle button groups in the design system, always wrap related toggles in a `role="group"` container with an `aria-label`. Individual toggle buttons must use `aria-pressed` to indicate state. Furthermore, ensure all interactive elements receive explicit `focus-visible:ring-2 focus-visible:ring-brand focus-visible:outline-none` classes to guarantee distinct keyboard focus rings, and hide decorative/redundant icons from screen readers using `aria-hidden="true"`.
