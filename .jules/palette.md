## 2024-05-18 - Accessible Toggle Groups and Keyboard Navigation in Dark Theme
**Learning:** Found that custom instrument selectors lacked the appropriate `role="group"` and `aria-label` to be grouped together by screen readers, and individual buttons were missing `aria-pressed` to indicate their toggle state. Additionally, Lucide icons next to text in buttons were redundantly read by screen readers without `aria-hidden="true"`, and the dark theme masked default browser keyboard focus outlines on interactive elements.
**Action:** Always wrap toggle button sets in a `role="group"` with an `aria-label`, use `aria-pressed` for their active states, apply `aria-hidden="true"` to decorative/redundant Lucide icons, and enforce visible focus states using `focus-visible:ring-2 focus-visible:ring-brand focus-visible:outline-none` across all custom interactive controls in the application's Tailwind dark theme.
