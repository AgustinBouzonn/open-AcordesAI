## 2025-01-28 - Missing Semantics and Focus Indicators on Icon-Only Buttons
**Learning:** Found a systemic pattern in `SongViewer.tsx` where icon-only action buttons (Share, Favorite, Comments) lacked both accessible names (`aria-label` or `title`) and visible focus indicators for keyboard navigation (`focus-visible` styles). This renders them completely inaccessible to screen reader users and keyboard-only navigators.
**Action:** Always add contextual `aria-label` and `title` attributes (in Spanish, matching the app's locale) to icon-only buttons. Add `aria-hidden="true"` to the decorative child Lucide icons. For keyboard accessibility in this dark theme, consistently apply `focus-visible:ring-2 focus-visible:ring-brand focus-visible:outline-none` classes to these interactive elements.
