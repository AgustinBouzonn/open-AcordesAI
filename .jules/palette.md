## 2024-04-10 - Accessible Interactive Icon Controls
**Learning:** In highly interactive components like `SongViewer`, icon-only buttons (like share, favorite, decrease/increase font size, decrease/increase transpose, decrease/increase autoscroll speed) often lack screen reader accessibility, relying entirely on visual cues. Furthermore, dynamic state readouts (like the current transpose level or autoscroll speed) are updated visually but may not be announced by screen readers.
**Action:** Always provide explicit `aria-label` or `title` attributes (preferably in the app's localized language, e.g., Spanish) on icon-only buttons. Apply `focus-visible:ring-2 focus-visible:ring-brand focus-visible:outline-none` for proper keyboard focus indication on dark themes. For dynamic text readouts that update inline without page reloads, use `aria-live="polite"` so screen readers properly announce state changes to users.
