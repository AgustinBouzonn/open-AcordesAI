## 2024-05-20 - Stateful Icon-Only Buttons and Keyboard Focus Accessibility
**Learning:** In dark-themed React applications with stateful icon-only buttons (like Favorite toggles), standard `:focus` often blends with the background or lacks visual clarity. Screen readers also need dynamic `aria-label`s rather than static descriptions to understand the action that will occur. Additionally, decorative icons inside these buttons (e.g., Lucide React icons) are redundantly announced unless explicitly hidden.
**Action:** When creating stateful icon-only buttons, always apply explicit keyboard focus classes (e.g., `focus-visible:ring-2 focus-visible:ring-brand focus-visible:outline-none`), use dynamic `aria-label` and `title` attributes that reflect the current/next state (e.g., 'Añadir a favoritos' vs 'Quitar de favoritos'), apply `aria-expanded` for toggles, and ensure the child icon components include `aria-hidden="true"`.
